# 领地系统

如果你准备长期玩，圈地是最应该尽早学会的功能。它能保护你的建筑、箱子、农场和机器，避免因为偷窃、误拆或恶意破坏造成损失。

> **一句话建议**
> 找到家以后，先圈地，再装修。对长期玩家来说，圈地比多挖一组钻石更重要。

## 为什么要圈地

领地系统可以保护：

- 房子、仓库和装饰建筑
- 箱子、熔炉、漏斗等容器
- 农场、刷怪塔、红石机器
- 你和朋友长期建设的基地成果

没有圈地的地方不代表一定会出事，但一旦被破坏，管理虽然可以查日志，处理和恢复都会更麻烦。

## 新手圈地流程

### 第一步：准备圈地工具

通常使用 **木锄** 作为圈地工具。具体以服务器提示为准。

### 第二步：选择两个对角

领地是一个长方体区域。你需要选中这个区域的两个对角：

- 左键选择第一个角
- 右键选择另一个对角

你可以把它理解成：用两个点框住一整个立体盒子。

### 第三步：创建领地

输入：

```text
/res create 领地名
```

例如：

```text
/res create home1
```

创建成功后，系统会提示领地已经建立。

## 圈地时最容易漏的地方

新手常见问题不是不会圈地，而是圈得太小。建议检查这些位置有没有被保护进去：

- 地下仓库
- 地下农场
- 红石机器层
- 出入口和楼梯井
- 传送点周围
- 箱子墙背后区域

如果你的基地有地下部分，记得把高度也一起考虑进去。

## 给朋友权限

默认情况下，领地通常只允许领地主人建造和交互。如果你和朋友一起玩，可以给对方单独开权限。

### 常见权限含义

| 权限 | 作用 | 适合给谁 |
| :--- | :--- | :--- |
| `build` | 放置和破坏方块 | 可信队友 |
| `container` | 打开箱子、熔炉等容器 | 共享仓库成员 |
| `use` | 使用门、按钮、拉杆等 | 访客或队友 |
| `tp` | 传送到领地 | 常来基地的朋友 |

### 指令写法

```text
/res pset 领地名 玩家名 权限 true
```

例如：

```text
/res pset home1 Steve build true
```

意思是允许 `Steve` 在 `home1` 领地内建造。

如果要取消权限，把最后的 `true` 改成 `false`。

## 不想手打权限怎么办

如果服务器支持图形化权限面板，可以输入：

```text
/res set
```

图形界面通常更直观，也更不容易输错。新手建议优先用界面确认权限状态。

## 常用维护指令

| 指令 | 用途 |
| :--- | :--- |
| `/res info` | 查看当前所在领地信息 |
| `/res tp [领地名]` | 传送到指定领地 |
| `/res set` | 打开权限管理界面 |
| `/res remove [领地名]` | 删除领地 |

删除领地前请确认里面没有重要建筑或共享成员。删错领地可能导致保护立即失效。

## 子领地适合什么时候用

子领地是在大领地里再划出一小块单独管理的区域。适合这些情况：

- 公会基地里给成员分房间
- 主基地里划出商店区
- 仓库、机器房、公共区域分开权限
- 只给朋友开放一小块区域，而不是整个基地

如果你的基地只是一座普通小房子，一开始不需要急着学子领地。等规模变大后再用也可以。

## 常见问题

### 领地一定要很大吗？
不一定，但至少要覆盖你当前最重要的建筑、箱子和机器。前期可以先保护核心区域，后期再扩建。

### 圈地后别人还能进来吗？
要看服务器默认权限。有些领地只禁止破坏和开箱，但仍允许进入。你可以通过 `/res set` 查看和调整。

### 没圈地被偷了怎么办？
可以联系管理尝试调查，但未圈地区域处理难度更高，也不一定能完整恢复。以后请尽快圈地。

### 我能给陌生人箱子权限吗？
不建议。`container` 权限意味着对方可能打开你的箱子、熔炉等容器。只给你真正信任的人。

## 推荐做法

建家后的顺序建议是：

1. 找到合适位置
2. 设置家 `/sethome base`
3. 圈地保护核心区域
4. 把仓库和机器放进保护范围
5. 再邀请朋友一起建设

这样可以大幅减少后续纠纷。
