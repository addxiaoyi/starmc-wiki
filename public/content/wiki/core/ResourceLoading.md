# 自定义资源加载

如果你进服后发现方块、物品、图标或字体显示不正常，第一件事通常就是检查资源包。服务器很多特色内容都依赖它。

## 先看结论

建议你把服务器资源包设为：

- **启用**
- 或 **提示**

这样最不容易错过自定义内容。

## 资源包是干什么的

你可以把它理解成：服务器发给客户端的一套“额外显示内容”。

它负责让你看到：

- 自定义家具
- 作物和植物外观
- 称号和 UI 图标
- 一些特殊物品的材质

如果资源包没加载好，这些内容就可能看起来不对。

## 推荐怎么设置

### 进服前

如果启动器或者客户端有相关设置，建议提前确认：

- 没有强制关闭资源包
- 没有加载冲突的本地材质包
- 客户端版本尽量使用推荐版本

### 进服后

进入服务器时如果弹出资源包提示，建议直接允许。

如果你已经在服内但显示异常，可以先：

1. 退服
2. 重新进服
3. 再看是否恢复正常

## 常见问题

### 物品变成紫黑方块

这通常说明资源包没有成功加载。

你可以先试：

- 退出再重新进入服务器
- 检查本地是否有冲突材质包
- 按公告或指引手动重新获取资源包

### 聊天框或 UI 出现方块字

这通常是字体或本地材质包和服务器资源冲突了。

建议先关闭本地带有特殊字体的材质包，再重进试试。

### 看不到自定义图标

这也是资源包没有正常生效的常见表现。通常重进或重新加载后就会恢复。

## 一个更稳的习惯

如果你平时喜欢装很多材质包，建议把服务器资源和本地材质分开管理。这样出问题时更容易判断到底是谁冲突了。

## 你可以这样排查

1. 看服务器有没有资源包提示
2. 确认你是否点了“允许”
3. 退出并重进一次
4. 暂时关闭本地材质包
5. 再检查显示是否恢复

## 常用指令

| 指令 | 功能 |
| :--- | :--- |
| `/ce pack` | 获取当前资源包的下载链接 |
| `/ce reload` | 管理员强制重新加载资源引擎 |
